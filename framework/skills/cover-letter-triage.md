# Skill: Cover Letter Triage

**Load when:** deciding whether to write a cover letter for a given application,
or drafting one that's already been decided as worth writing.

**Depends on:** `framework/skills/resume-lane-strategy.md` for which lane's
framing to lead with.

---

## The problem

A cover letter is not free. Writing one for every application, on the
assumption that more effort per application is strictly better, misallocates
effort in a way that's easy to miss without deliberately checking: a cold
application to a large company is read, if at all, by a recruiter doing a fast
pass against a resume, and a cover letter rarely changes that outcome. The same
effort spent on a warm introduction, or on a role where a human actually reads
the letter, converts at a meaningfully different rate.

**A cover letter is not a default step.** Decide whether to write one before
drafting, not after.

## Triage

- **Cold application, larger organization, no cover-letter field:** skip it.
  Redirect the effort toward a referral path or a direct message to a hiring
  manager instead, which converts better at this org size than a cold letter.
- **Cold application, larger organization, letter optional:** usually still
  skip, unless there's a genuine organization-specific hook or a warm angle
  worth naming.
- **Letter required by the application:** write a short, specific one. Don't
  pad it to fill space just because a field exists.
- **PC roles (see `resume-lane-strategy.md`):** always write one, and make
  it the most voice-driven, specific version possible. At this org size a human,
  often the actual hiring decision-maker, is likely to read it, and it can
  meaningfully shape how the role is perceived to fit. This is where letter
  effort has the highest return.
- **Essay or application-specific questions:** treat as the real artifact, not
  a formality. These get read, unlike many attached letters. Answer
  specifically and in the candidate's own voice, to the stated length.

## When a letter is worth writing

- Lead from the same verified facts and the same lane framing as the resume
  being submitted alongside it. See `resume-lane-strategy.md` and
  `human-writing-standards.md`.
- Structure: a hook specific to the actual organization's actual problem, then
  two or three proof points mapped to the role, then a short close. Avoid
  generic openers that could be pasted into any application unchanged.
- Run the human-writing standards check before presenting any draft as final.

## Artifacts that often outperform a cover letter

- **A short, direct message to a person with a warm connection** to the role
  outperforms a formal letter nearly every time it's available as an option.
- **A short standalone document showing how the candidate thinks** (a
  philosophy statement, a design note, anything substantive that isn't a
  cover letter) can be a stronger differentiator for organizations that give an
  open-ended upload field, specifically because almost no other applicant
  submits one.
