# Skill: Warm Outreach

**Load when:** working any thread involving a referral, a former colleague, a
hiring manager, or a warm contact — or deciding whether to invest in one over
another cold application.

**Depends on:** the private instance's verified experience summary for every
claim, and its outreach queue / company-brief files for state.

---

## Why this gets its own skill

Warm paths convert at a categorically different rate than cold applications,
and they carry a different kind of risk: a contact's reputation rides on the
outcome. Cold-application methodology (triage, forms, tracking) doesn't
transfer — this channel needs its own sequencing rules and its own hygiene.

## Sequencing rules

1. **Apply first, then message**, when the role already exists as posted.
   Same-day application timing has real value on boards that badge early
   applicants, and "my application is in" is a stronger opener than "should
   I apply?" The exception: when the contact might reshape the role itself
   (small-org leadership lane), talk before applying.
2. **When a platform blocks messages to distant connections**, send the bare
   connection request and queue the message in a pending-outreach state file
   — so the moment they accept, the message goes out without re-drafting.
3. **A gated role plus a warm contact = raise the gate in conversation.** A
   knockout form question and a warm conversation are different instruments
   for the same gap: a years-of-X dropdown will reject a transferability
   argument that lands fine with a human. Answer forms honestly, and make
   the argument where it has room to land.
4. **Close loops.** After any application a contact helped with, tell them
   directly — it activates whatever they offered.

## Message shape

Open with the specific role or shared context; one concrete credential
mapped to their problem, traced to the verified experience summary; name
real gaps plainly (it reads as credible, not weak); one clear ask; short.
Run the human-writing gate before sending.

## Hygiene

- Never let a contact vouch for a claim that isn't in the verified summary.
- Track every sent-and-waiting thread in one queue file, with the queued
  message stored verbatim — re-drafting from memory reintroduces drift.
- Before any call, pull (or create) the per-company brief; after, update it
  and log the outcome the same day. See the screen-prep skill.
